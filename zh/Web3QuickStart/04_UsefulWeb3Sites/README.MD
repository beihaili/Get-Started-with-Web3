## 一、区块链浏览器

区块链浏览器是 Web3 世界最基础的“搜索引擎”。遇到任何交易、地址、合约、代币或 NFT 问题，第一步通常不是去问客服，而是先查链上记录。

| 链/生态 | 推荐工具 | 主要用途 |
|---------|----------|----------|
| Ethereum | [Etherscan](https://etherscan.io/) | 查交易、地址、合约源码、事件日志、授权 |
| Base | [Basescan](https://basescan.org/) | 查 Base 上的交易、合约和代币 |
| Arbitrum | [Arbiscan](https://arbiscan.io/) | 查 Arbitrum One 交易和合约 |
| Optimism | [Optimistic Etherscan](https://optimistic.etherscan.io/) | 查 OP Mainnet 交易和合约 |
| Polygon | [PolygonScan](https://polygonscan.com/) | 查 Polygon PoS 生态 |
| BNB Chain | [BscScan](https://bscscan.com/) | 查 BNB Chain 交易和代币 |
| Solana | [Solscan](https://solscan.io/) / [Solana Explorer](https://explorer.solana.com/) | 查 Solana 交易、账户、程序和 SPL 代币 |
| Bitcoin | [mempool.space](https://mempool.space/) | 查比特币交易、区块、费率和 mempool 状态 |

新手要重点学会看 5 个字段：

1. **交易状态**：Success、Fail、Pending 分别代表什么。
2. **From/To**：谁发起、发给谁或调用了哪个合约。
3. **Value**：是否直接转出原生资产。
4. **Input Data**：合约调用的具体内容。
5. **Token Transfers**：是否发生了 ERC-20、NFT 或 SPL 代币转移。

## 二、数字资产行情网站

行情网站不是投资建议来源，而是基础数据入口。

| 工具 | 适合查看 | 注意事项 |
|------|----------|----------|
| [CoinMarketCap](https://coinmarketcap.com/) | 市值排名、交易所、基础行情 | 新币和小币数据需要交叉验证 |
| [CoinGecko](https://www.coingecko.com/) | 市值、分类、交易对、开发者信息 | 适合和 CoinMarketCap 对照 |
| [TradingView](https://www.tradingview.com/) | K 线、技术指标、宏观市场 | 不是链上数据源 |
| [Token Terminal](https://tokenterminal.com/) | 协议收入、费用、基本面 | 更适合研究协议业务质量 |
| [Artemis](https://app.artemis.xyz/) | 公链活跃用户、交易数、费用 | 适合横向比较生态数据 |

看行情时不要只看价格，还要看：

- 交易量是否真实。
- 流动性在哪些交易所。
- 市值和完全稀释估值差距。
- 代币释放计划。
- 协议收入是否支撑估值。

## 三、DeFi 工具

DeFi 工具用于理解协议、收益和风险。

| 工具 | 主要用途 |
|------|----------|
| [DefiLlama](https://defillama.com/) | TVL、链上协议分类、收益、稳定币、桥资金流 |
| [Dune](https://dune.com/) | 社区 SQL 看板，适合看链上行为分析 |
| [DeBank](https://debank.com/) | 查看多链地址资产、授权和 DeFi 仓位 |
| [Zapper](https://zapper.xyz/) | 多链资产聚合和地址画像 |
| [Revoke.cash](https://revoke.cash/) | 检查和撤销代币授权 |
| [L2BEAT](https://l2beat.com/) | 查看 L2 TVL、风险、数据可用性和排序器状态 |

使用 DeFi 数据时要避免一个误区：TVL 高不等于安全，收益高不等于值得参与。你还要看审计、治理权限、预言机、清算机制、稳定币抵押物和退出流动性。

## 四、靠谱的交易所网站

中心化交易所适合入金、出金和基础交易，但它不是链上钱包。交易所账户里的资产由交易所托管，和你自己掌握私钥的钱包不是一回事。

选择交易所时建议检查：

- 是否长期运营且公开披露储备证明。
- 是否支持你所在地区的合规要求。
- 提币网络是否清晰标注。
- 小额提币是否顺畅。
- 是否开启 2FA、防钓鱼码和提现白名单。

常见交易所包括 [Binance](https://www.binance.com/)、[OKX](https://www.okx.com/)、[Coinbase](https://www.coinbase.com/) 等。不同地区可用服务不同，实际使用前要确认当地法规和平台条款。

| 任务 | 工具 | 使用场景 |

| 节点和 RPC | Alchemy、Infura、QuickNode、Ankr | 读取链上数据、广播交易 |

### L2 与跨链

| 任务 | 工具 | 使用场景 |

### 研究与信息源

| 任务 | 工具 | 使用场景 |
|------|------|----------|
| 协议文档 | ethereum.org、Bitcoin Optech、项目官方 Docs | 学习一手资料 |

## 常见问题

#### ❓ 区块链浏览器和普通网页浏览器有什么不同？

区块链浏览器和普通网页浏览器的主要区别在于：

1. ** 搜索内容 **：普通浏览器搜索网页，区块链浏览器搜索交易、区块、地址和合约。
2. ** 目标 **：普通浏览器展示网络内容，区块链浏览器用于查看区块链上的数据。
3. ** 功能 **：区块链浏览器提供特殊功能，如交易确认查询、智能合约代码查看、Gas 费计算等。

#### ❓ 为什么需要使用多个加密货币行情网站？

使用多个加密货币行情网站的原因包括：

1. ** 数据准确性 **：不同网站的数据来源和更新速度不同，交叉验证能获得更准确的信息。
2. ** 功能互补 **：比如 CoinMarketCap 与 CoinGecko 的 API 使用放松程度不同，某些数据处理更全面。
3. ** 市场角度 **：不同平台对新项目的收录标准和评级方式不同，提供多角度观点。

#### ❓ 如何判断交易所是否靠谱？

判断交易所靠谱性的要点包括：

1. ** 监管状态 **：是否获得合法持牌牌照和监管许可。
2. ** 市场历史 **：运行时间与运行记录，包括安全遇到黑客攻击时的处理方式。
3. ** 交易量和流动性 **：高交易量通常意味着更多用户信任。
4. ** 透明度 **：是否公开其资金储备和公司结构。
5. ** 用户评价 **：社区中的口碑和用户体验。
