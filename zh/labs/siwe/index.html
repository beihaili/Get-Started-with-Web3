这个实验讲什么

Nonce 应由可信 relying party 生成并一次性使用，避免旧签名被再次播放。

Domain 和 URI 把消息绑定到发起签名的网站，帮助钱包和用户识别钓鱼请求。

Issued At 和 Expiration Time 缩小签名认证消息的有效时间窗口。

签名不是 session。生产级 SIWE 需要服务器验证签名，并创建、轮换、失效化 session。

不是生产级认证

这个静态 GitHub Pages demo 不签发可信 nonce、不存储 session，也不验证 ERC-1271 合约钱包签名。它只是用来学习 SIWE 消息结构。

与学习身份的关系

未来同样的模式可以帮助学习者在领取徽章、证书或 verifiable credential 前证明钱包控制权。当前项目还不声称已经上线真实账户系统。
