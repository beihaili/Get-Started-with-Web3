MVP 范围

  • 连接和断开钱包
  • 展示账户、链名称和 chain ID
  • 在批准的网络集合中切换
  • 签名一条教学消息，并在本地查看结果

这个实验讲什么

EIP-6963 让多个注入式钱包主动声明自己，页面可以展示钱包选择，而不是假设只有一个全局 window.ethereum provider。

消息签名只证明当前提示下的钱包控制权。它不会发送交易、移动资产，也不会创建生产级登录会话。

浏览器钱包没有统一的通用断开 API。本实验只清空本地状态，钱包侧授权仍由用户在钱包 UI 中管理。

隐私边界

实验室不会把原始钱包地址、签名、已签消息、provider 对象或交易 payload 发送到 analytics。账户和签名只在当前标签页展示，地址在 UI 中会被缩略。

下一步实现切片

把这个无依赖的 EIP-6963/EIP-1193 切片作为基线。只有后续实验确实需要抽象层时，再引入 wagmi、viem、RainbowKit 或 Reown AppKit。
