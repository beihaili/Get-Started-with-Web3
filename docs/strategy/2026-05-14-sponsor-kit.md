## One-Line Pitch

Get Started with Web3 is an open-source, bilingual, AI-native Web3 curriculum for beginners, builders, and agents.

## Why Sponsor

This project is a long-lived educational asset, not a short campaign. Sponsoring it gives a Web3 product credible placement inside a practical learning path that covers wallet onboarding, Bitcoin, DeFi, Layer 2, DAO, smart accounts, security, and builder labs.

The audience is especially valuable for:

- Wallets and account abstraction products that need safer onboarding.
- RPC, infra, API, explorer, and devtool products that need new builders.
- Security products that want to reach beginners before they form bad habits.
- L2 ecosystems that need clear developer and user education.
- Exchanges or on-ramp providers that can meet a strict trust and disclosure policy.

## Placement Rules

Sponsor placements must preserve learner trust.

- Paid placements are disclosed.
- Links use appropriate `rel` attributes.
- Sponsor copy stays short and factual.
- No intrusive popups, blocking modals, autoplay, or misleading CTAs.
- Sponsors cannot influence lesson conclusions or security guidance.
- The project owner may reject or remove a sponsor if the product becomes unsafe or reputationally risky.

## Sponsor Acceptance Policy

Allowed:

- Wallets with clear security posture.
- Infrastructure and API providers.
- Block explorers and analytics tools.
- Security tooling and audit education.
- L2 ecosystems and developer platforms.
- Education, grant, and public-goods programs.

Requires extra review:

- Exchanges and on-ramp providers.
- Yield, staking, restaking, leverage, or structured products.
- Token launch platforms.
- NFT or game projects.
- Products targeting beginners with financial incentives.

Rejected by default:

- Opaque yield products.
- High-pressure trading products.
- Projects with unresolved scam accusations.
- Products that require users to reveal seed phrases or unsafe wallet permissions.
- Anything that conflicts with beginner safety.

## Sponsor Metrics To Report Monthly

Minimum report:

- GitHub stars and monthly star growth.
- GitHub repo views and unique visitors.
- Public site users/pageviews if available.
- Top viewed pages if available.
- New lessons, tools, or AI-native artifacts shipped.
- Contributor and PR activity.

Do not overstate metrics. If a number is from GitHub traffic and only covers the repository, label it as GitHub traffic.
