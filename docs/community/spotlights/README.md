# Contributor Spotlights

Contributor spotlights turn public contributions into durable recognition, onboarding proof, and shareable community updates. They should cite merged PRs, closed issues, and public project metrics only.

## Operating Rules

- Highlight public GitHub usernames, PRs, issues, releases, or posts.
- Ask for consent before using personal details beyond public profile information.
- Keep praise tied to concrete learner, contributor, or project impact.
- Do not promise token rewards, paid placement, sponsor benefits, financial upside, or preferential review.
- Link the contributor ladder and good-first issue queue so new contributors have a clear next step.
