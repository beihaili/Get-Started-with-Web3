What this lab teaches

A nonce must be generated by a trusted relying party and used once so an old signature cannot be replayed later.

The domain and URI bind the message to the site requesting the signature, helping wallets and users detect phishing.

Issued-at and expiration timestamps narrow the valid time window for a signed authentication message.

A signature is not a session. Production SIWE needs a server to verify the signature and create, rotate, and invalidate sessions.

Not production authentication

This static GitHub Pages demo does not issue trusted nonces, store sessions, or verify ERC-1271 contract-wallet signatures. It is a learning surface for SIWE message anatomy.

Learning identity connection

The same pattern can later help a learner prove wallet control before receiving a badge, certificate, or verifiable credential. This project does not claim that account system yet.
