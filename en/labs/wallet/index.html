Wallet Interoperability Lab

A focused lab surface for wallet discovery, network context, and message signing. The first slice is intentionally static while the wallet stack is reviewed.

Current status

Scaffold ready. Wallet dependencies are not installed yet, so no wallet connection, signing, transaction, or authentication flow runs on this page today.

MVP scope

  • Connect and disconnect wallet
  • Show account, chain name, and chain ID
  • Switch between an approved set of networks
  • Sign one educational message and inspect the result locally

Privacy boundary

The lab must never send raw wallet addresses, signatures, signed messages, provider objects, or transaction payloads to analytics.

Next implementation slice

Add route tests and keep the page dependency-free. Install wagmi, viem, RainbowKit, or Reown AppKit only in the follow-up implementation PR.
