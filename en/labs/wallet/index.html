What this lab teaches

EIP-6963 lets multiple injected wallets announce themselves so a page can offer a wallet choice instead of assuming one global window.ethereum provider.

A message signature proves control of a wallet for this prompt only. It does not send a transaction, move assets, or create a production login session.

Browser wallets do not share one universal disconnect API. This lab clears local state and leaves wallet-side permissions under the user's wallet UI.

Privacy boundary

The lab never sends raw wallet addresses, signatures, signed messages, provider objects, or transaction payloads to analytics. Account and signature values are displayed only in this tab, and addresses are shortened in the UI.

Next implementation slice
